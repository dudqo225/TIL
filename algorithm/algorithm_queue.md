### 1. 큐

#### 큐(Queue)의 특성

- 스택과 마찬가지로 삽입/삭제의 위치가 제한적인 자료구조
- 큐의 뒤에서는 삽입(`push`)만 하고, 큐의 앞에서는 삭제(`pop`)만 이루어지는 구조
- **선입선출(FIFO, First In First Out)**

#### 큐의 선입선출 구조 & 연산

- 구조
  - 머리 (Front) : 저장된 원소 중 첫번째 원소
  - 꼬리(Rear) : 저장된 원소 중 마지막 원소

- 기본 연산
  - 삽입 : enQueue
  - 삭제 : deQueue

- 주요 연산
  - `enQueue(item)` : 큐의 rear 다음에 원소를 삽입하는 연산
  - `deQueue()` : 큐의 front에서 원소를 삭제하고 반환하는 연산
  - `createQueue()` : 공백 상태의 큐를 생성하는 연산
  - `isEmpty()` : 큐가 공백상태인지 확인하는 연산
  - `isFull()` : 큐가 포화상태인지 확인하는 연산
  - `Qpeek()` : 큐의 front에서 원소를 삭제없이 반환하는 연산

#### 큐의 구현

##### 선형 큐

- 1차원 배열을 이용
  - 큐의 크기 = 배열의 크기
  - front : 저장된 첫번째 원소의 인덱스
  - rear : 저장된 마지막 원소의 인덱스
- 상태 표현
  - 초기 상태 : front == rear == -1
  - 공백 상태 : front = rear
  - 포화 상태 : rear = n-1 (n은 배열의 크기, n-1은 배열의 마지막 인덱스)

- 선형큐 이용시 문제점
  - 잘못된 포화상태 인식
    - 선형큐를 이용해서 원소의 삽입/삭제를 반복할 경우, 배열 앞부분에 활용가능한 공간이 있음에도 불구하고, rear = n-1 포화상태로 인식해서 더 이상 삽입을 수행하지 않음
  - 해결방법 1
    - 매 연산이 이루어질때마다 저장된 원소들을 배열 앞부분으로 이동시킨다.
    - but, 원소 이동에 많은 시간이 소요되어 큐의 효율성이 떨어진다.
  - 해결방법 2
    - 1차원 배열을 사용하되, 논리적으로 배열의 처음-끝이 연결된 원형 형태의 큐를 이룬다고 가정하고 사용
    - Index의 순환
      - front, rear 위치가 배열의 마지막 인덱스인 n-1을 가리킨 후, 논리적 순환을 위해 배열의 처음 인덱스인 0으로 이동해야 한다.
      - 나머지 연산자 `mod`를 사용
    - front 변수
      - 공백 상태와 포화 상태의 구분을 쉽게하기 위해서 front가 있는 자리는 사용하지 않고 항상 빈자리로 둔다.

##### 연결 큐

- 단순 연결 리스트(Linked List)를 이용한 큐
  - 큐의 원소 : 단순 연결 리스트의 노드
  - 큐의 원소 순서 : 노드의 연결 순서. 링크로 연결되어 있다.
  - front : 첫번째 노드를 가리키는 링크
  - rear : 마지막 노드를 가리키는 링크

### 2. 우선순위 큐

#### 우선순위 큐 (Priority Queue)

- 특성
  - 우선순위를 가진 항목들을 저장하는 큐
  - FIFO 순서가 아닌, 우선순위가 높은 순서대로 먼저 나가게 됨
- 적용 분야
  - 시뮬레이션 시스템
  - 네트워크 트래픽 제어
  - OS의 테스크 스케쥴링
- 구현
  - 배열을 이용
    - 가장 앞에 최고 우선순위의 원소가 위치하게 됨
    - 삽입/삭제 연산이 일어날 때 원소의 재배치가 발생 → 시간 소요, 메모리 낭비가 큼
  - 리스트를 이용

<br>

### 3. 버퍼

#### 버퍼(Buffer) 란?

- 데이터를 한 곳에서 다른 곳으로 전송하는 동안 일시적으로 그 데이터를 보관하는 메모리의 영역
- **버퍼링** : 버퍼를 활용하는 방식 or 버퍼를 채우는 동작을 의미
- 버퍼의 자료 구조
  - 버퍼는 입출력 및 네트워크와 관련된 기능에서 이용된다.
  - 순서대로 입력-출력-전달 되어야하기 때문에 FIFO 방식의 큐가 활용됨

<br>

### 4. BFS

#### 너비우선탐색(Breadth First Search, BFS)

- 시작 노드의 인접 노드들을 먼저 모두 차례로 방문한 후, 방문했던 정점을 시작점으로 해서 다시 인접 정점들을 차례로 방문하는 방식
- FIFO 형태의 자료구조 큐를 활용함

```python
# BFS 알고리즘
def BFS(G, v):                  # 그래프 G, 탐색 시작노드 v
    visited = [0] * n           # n: 정점의 개수
    queue = []                  # 큐 생성
    queue.append(v)             # 시작점 v를 큐에 삽입
    while queue:                # 큐가 비어있지 않은 경우
        t = queue.pop(0)        # 큐의 첫번째 원소를 반환
        if not visited[t]:      # 방문하지 않은 곳이라면
            visited[t] = True   # 방문한 것으로 표시
            visit(t)
        for i in G[t]:          # t와 연결된 모든 간선에 대해
            if not visited[i]:  # 방문하지 않은 곳이라면
                queue.append(i) # 큐에 넣기
```
